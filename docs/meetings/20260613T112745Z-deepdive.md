## External Signals (Researcher)

- Streamlit and Jupyter can run in parallel because they are separate local processes. Use Streamlit on port `8501` and Jupyter on port `8888`; switch to `8502` or `8889` if a port is busy.
- Full-data app and notebook runs duplicate memory and preprocessing work. For classroom demos, prepare capped data with `python scripts/prepare_real_data.py --max-rows 5000` or `--max-rows 10000`.
- Streamlit caching is the right pattern for dataframe loading and feature preparation, but cache is process-local and does not share work with Jupyter.
- Headless Streamlit testing exists, but a practical near-term layer is testing app helper functions plus a startup smoke check.
- Avoid fragile pandas APIs in demo code. Use explicit numeric selection before summary statistics.
